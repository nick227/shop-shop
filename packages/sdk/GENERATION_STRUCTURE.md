## File Categories

### рџ”ґ Generated Files (DO NOT EDIT)
- All files in `generated/` directory
- Files with `AUTO-GENERATED` headers
- Files generated by OpenAPI tools
- Files generated by custom scripts

### рџџў Source Files (EDITABLE)
- Files in `src/` directory

- Documentation files

## Generation Workflow

1. **OpenAPI Generation**: `npm run gen` в†’ `generated/sdk/`
2. **Frontend Types**: `npm run gen:frontend-types` в†’ `generated/frontend/types/`
3. **API Wrappers**: `npm run gen:wrapper` в†’ `generated/frontend/api/`
4. **React Hooks**: `npm run gen:hooks` в†’ `generated/frontend/hooks/`
5. **Build**: `npm run build` в†’ `dist/`

## Import Strategy

### From Apps (apps/web/src/):
```typescript
// Generated API wrappers
import { apiWrapper } from '@packages/sdk/generated/frontend/api/apiWrapper'

// Generated types
import type { StoreResponse } from '@packages/sdk/generated/frontend/types/backend-types'

// Generated hooks
import { useStores } from '@packages/sdk/generated/frontend/hooks/generated'

// Source utilities
import { createApiClient } from '@packages/sdk'
```

### From SDK Package:
```typescript
// Re-export generated files
export * from './generated/frontend/api/apiWrapper'
export * from './generated/frontend/types/backend-types'
export * from './generated/frontend/hooks/generated'

// Source files
export * from './src/realtime'
```

## Benefits

1. **Clear Separation**: Generated files are clearly separated from source files
2. **No Confusion**: Developers know what they can and cannot edit
3. **Better Organization**: Related generated files are grouped together
4. **Easier Maintenance**: Generation scripts have clear output targets
5. **Version Control**: Generated files can be easily ignored or tracked as needed

## Migration Steps

1. Move existing generated files to new structure
2. Update generation scripts to output to new locations
3. Update import paths in apps
4. Update package.json exports
5. Update .gitignore patterns
